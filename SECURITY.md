# Security Policy

## Supported Versions

MindNoron is pre-1.0 software. Security fixes should target the latest commit
on `main` unless a release branch is created later.

## Reporting A Security Issue

Please do not open a public issue for sensitive security reports.

If you find a vulnerability, contact the maintainer through the GitHub profile
or repository contact channel. Include:

- the affected version or commit;
- the operating system;
- clear reproduction steps;
- the impact you believe the issue has;
- any relevant logs or screenshots that do not expose private user data.

## Data Safety Notes

MindNoron stores personal data locally and can export JSON backups. Treat those
backup files as sensitive. Backups are not encrypted by the app today.
